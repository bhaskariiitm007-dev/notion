Installation Guide — All Three Platforms
Step-by-step for Mac, Windows, and Linux — follow the one for your system
🍎
macOS
Install Node.js from nodejs.org (LTS version)
Open Terminal app
Run install command
npm install -g @anthropic-ai/claude-code
Verify with: claude --version
Run claude to launch in any folder
🪟
Windows
Install Node.js from nodejs.org (LTS)
Download Windows Terminal (from Microsoft Store)
Set PATH if install complains about path
npm install -g @anthropic-ai/claude-code
If path error: add npm global to system PATH
Run: claude --version to verify
🐧
Linux
Install Node.js via package manager (nvm recommended)
Use nvm for version management
npm install -g @anthropic-ai/claude-code
Add npm global to PATH in .bashrc/.zshrc
Perfect for server deployments as well
Works identically on remote Linux servers

Authentication Methods
Two ways to connect Claude Code to the Claude brain
1
Claude.ai Subscription (easiest) — run claude, login with your account. Pro plan includes Sonnet access. Most beginners should use this.
2
API Key (pay-per-token) — get a key from console.anthropic.com. Set as ANTHROPIC_API_KEY. Full model access, charged per token.
✓
Adding extra credits — even on subscription, you can load $20 credit for extra API calls if you're building heavily. Prevents hard stops.
✓
Check auth status — claude doctor checks your installation, auth, and connection. Run when something seems wrong.

Running in VS Code
The best development experience — see changes as they happen
1
Open VS Code — open your project folder in VS Code
2
Install extension — search "Claude Code" in Extensions tab. Install the official Anthropic one (11M+ installs).
3
Open Claude panel — click the Claude icon in the sidebar. Works just like the terminal version but shows file diffs.

Common Setup Issues & Fixes
The problems most people hit — and how to solve them
!
Windows PATH error — "claude is not recognized" means the npm global folder is not in PATH. Add %AppData%\npm to system PATH.
!
Node version too old — Claude Code requires Node 18+. If it fails, update Node via nvm or the official installer.
!
Permission denied (Mac/Linux) — don't use sudo for npm. Use nvm to install Node instead, which avoids permission issues.
!
Auth loop — if login keeps redirecting, clear browser cookies, try incognito, or use the API key method instead.
✓
Ultimate fix — run claude doctor. It checks installation health and tells you exactly what needs fixing.
