⭐ THE KEY INSIGHT
An LLM is trained like a student who studied for an exam. When you ask it something, it predicts the most likely next word, then the next, building up a complete, coherent answer.

The Claude Model Family — Speed vs Intelligence
Three tiers designed for different jobs — pick the right tool for each task
TIER 1 — FAST ⚡
Haiku
Speed First
Fastest response, lowest token cost. Great for simple tasks: quick questions, summaries, maintenance, error hunting.
TIER 2 — BALANCED ⚖️
Sonnet
Daily Driver
The everyday workhorse. Balances capability and cost for documents, coding, analysis, and most everyday tasks.
TIER 3 — MAXIMUM 👑
Opus
Deep Reasoning
Most capable. Complex planning, ambiguous requirements, architecture design. Use when it really needs to think hard.
⚡ Haiku = Maintenance⚖️ Sonnet = Building👑 Opus = Planning
🎯
Model Selection Strategy
The professional approach — Opus for plans, Sonnet for work, Haiku for checks
1
Start with Opus — use it for initial planning, architecture design, and understanding complex requirements. It thinks deepest.
2
Switch to Sonnet — for building major components, writing code, and day-to-day development work where balance matters.
3
Use Haiku for checks — maintenance, finding bugs, quick formatting, simple edits. Fast and cheap for these use cases.
!
Never always pick Opus — it's overkill for 90% of tasks and will burn through context and credits far too fast.
📋 Plan → Opus🔨 Build → Sonnet🔍 Fix → Haiku
🔬
Extended Thinking Mode
Claude deliberates before answering — depth over speed
★
What it is — Claude spends extra time reasoning through the problem before giving a final answer. Like "show your work".
✓
When to use — complex plans, ambiguous requirements, multi-step architecture, debugging tricky logic
✓
Context benefit — extended thinking uses a separate thinking tab, so it doesn't pollute your main context window
!
Don't overuse — unnecessary extended thinking wastes tokens. Only enable it when deeper reasoning genuinely matters.
🔬 Deep Reasoning✅ Separate Context⚠️ Use Judiciously
