Not a Chatbot — an Agent
Claude Code doesn't just generate text. It has a loop: plan, call tools, read results, plan again — until the task is done.
🔧
Tool Calls are the Power
Claude Code can read files, write code, run bash, search the web — all autonomously as part of one conversation.
💻
Terminal + IDE = Full Stack
Use it from the CLI for maximum power or inside VS Code / JetBrains for IDE-native AI pair programming.
🔄
The Claude Code Agentic Loop
How Claude Code thinks and acts — not one reply but a continuous plan-act-reflect cycle
💬
User Request
You describe the goal in natural language in the terminal or IDE
🧠
LLM Plans
Claude decides which tools to call and in what order to accomplish the task
🔧
Tool Execution
Tools run: read files, write code, bash commands, web search, edit functions
✅
Result + Loop
Claude reads results, re-plans if needed, and loops until the task is complete
⚡
⭐ THE KEY DIFFERENCE FROM CHAT
A regular LLM responds once. Claude Code loops — it calls tools, reads the output, updates its plan, and continues until the task is actually done. This is what "agentic" means.
⚖️
Claude (LLM Chat) vs Claude Code
Same brain, completely different execution model
💬 Claude (Chat / LLM)
You send a message, Claude replies once
No access to your file system
Cannot run code or bash commands
Text in → text out; pure generation
Like asking a smart person for advice
VS
⚡ Claude Code (Agentic)
Loops: plan → tool call → result → re-plan
Full read/write access to your project files
Runs bash, installs packages, tests code
Executes until the task is actually complete
Like a smart developer who also does the work

IDE Integration
VS Code, JetBrains, Cursor — Claude Code meets your editor
★
VS Code extension — install "Claude Code" from the marketplace; gives a side-panel chat with full file context
✓
JetBrains plugin — works in IntelliJ, PyCharm, DataGrip; same capability, different panel location
✓
Inline editing — select code, press the shortcut, ask Claude to rewrite, explain, or test it in place
✓
Context-aware — the IDE plugin passes your open files and errors automatically — no manual copy-paste
→
Terminal vs IDE: terminal gives more power and control; IDE gives comfort and visual feedback. Use both.
🖊️ VS Code ☕ JetBrains ⌨️ Terminal

Essential Claude Code Commands
The commands you'll use every single day
$
claude — launch interactive session in current directory; reads project files automatically
$
claude "task" — non-interactive one-shot: pass a task string and get the result without entering the REPL
$
/help — inside session: lists all slash commands available (skills, config, memory, etc.)
$
/init — creates a CLAUDE.md project config file in the current directory (covered in Chapter 4)
$
Ctrl+C — interrupt the current task if Claude is going in the wrong direction; always available
⌨️ CLI /help 🛑 Ctrl+C
