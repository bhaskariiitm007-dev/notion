The Claude Code Request Pipeline
What happens from the moment you press Enter to when code appears in your file
1 · USER
User Request — you type a natural language task in the terminal or IDE. E.g. "Create a PySpark script that reads from S3 and writes to Delta Lake."
2 · LLM
Claude (LLM) Plans — the LLM receives your message plus the current context window (files, history, CLAUDE.md). It decides which tool to call and with what parameters.
3 · TOOL CALL
Tool Execution — Claude emits a structured tool call (e.g. write_file with filename + content). The runtime executes it on your actual machine.
4 · RESULT
Result Injected — the output of the tool (file content, bash stdout, search results) is injected back into Claude's context window as the next message.
5 · LOOP
Re-plan or Finish — Claude re-reads the result, decides if more tools are needed, and loops. When done, it responds to you with a summary of what was accomplished.
💡
⭐ THE CORE INSIGHT
"Code generation" is just text generation. The LLM writes a string of code characters. The power comes from what happens next — Claude Code executes that text and feeds the result back in.

LangGraph Under the Hood
The framework powering Claude Code's agentic loop
📦 What LangGraph Provides
Graph of nodes (each = an LLM call or tool)
Directed edges (conditional branching)
Shared state across all nodes
Checkpoint & resume any step
🔬 How Claude Code Uses It
LLM node → decides action
Tool node → executes the action
State holds context window history
Edge: "done?" → exit, else → LLM again

Build Your Own (Light Version)
Understanding the architecture by recreating a minimal Claude Code
1
System prompt — describe available tools, their parameters, and when to use each one
2
Tool schema — define each tool as a JSON schema so the LLM knows exactly what to output
3
Execution loop — call the LLM, parse the tool call from the response, execute it, inject result back
4
Loop condition — if Claude emits a tool call → execute and loop; if it emits plain text → done
★
Why do this? — building a light version makes the "magic" of Claude Code transparent and de-mystifies agents
