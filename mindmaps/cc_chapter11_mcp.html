It's a Toolbox Standard
MCP isn't a Claude thing — it's an open standard built by Anthropic for all AI. Any AI (GPT, Perplexity, Cursor) can use MCP servers. Like HTTP for the web, MCP is the protocol for AI tools.
📦
Someone Else Maintains It
Before MCP, you'd write and maintain each API integration yourself. With MCP, Gmail/GitHub/Slack maintain their own server. You just connect and use — no maintenance, no API key juggling per-tool.
⚠️
Disable What You Don't Use
Every connected MCP server sends its tool definitions to Claude — even if you don't use it. 10 MCPs = 15k+ tokens burned before you type a word. Disable aggressively. Enables on demand.

Local vs Hosted Servers
Two ways an MCP server can run
💻 Local Server
Runs on your machine. A process Claude spawns locally. Talks to local files, databases, or APIs using your credentials. Perfect for personal data. Configured in settings.json or claude_desktop_config.json.
☁️ Hosted Server
Runs on the service's cloud (Gmail, GitHub, Slack). You connect via OAuth or API key. More secure — your data never touches a third machine. Simply click "Connect" in Claude.ai settings.
💡 Connecting is 1 click in Claude.ai → Settings → Integrations. For Claude Code terminal, add server config to .claude/mcp.json. Server handles auth — you never write API code.
⚠️
The Token Cost Warning
Unused MCPs silently eat your context budget
🚨 Each connected MCP sends tool definitions to Claude
Even if you never call it. Even if it's just connected. Each MCP server sends its full tool schema with every request. A single MCP can add 1-3k tokens. With 10 MCPs that's 15k+ tokens gone before you type "hi".
✓
Only connect what you need today — disconnect Gmail MCP when you're not doing email tasks
✓
Use /context to audit — run /context in Claude Code to see exactly how many tokens each MCP is using
✓
Toggle in settings — Claude.ai has quick enable/disable toggles. Turn off what you're not using.
✓
Think of it like RAM — each MCP is an open program. Close what you're not using right now.
