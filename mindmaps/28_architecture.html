LLM is the Brain
The LLM doesn't just respond — it decides which tool to call, with what inputs, and interprets the result to decide next steps.
🔧
Tools are the Hands
Built-in tools do the actual work: read files, write files, run bash commands, search code. LLM orchestrates, tools execute.
🔄
The Loop Never Stops
After each tool result, the LLM re-evaluates and decides: is the task done? If not, call another tool. This loop is the agent.

The Agentic Loop
The 4-phase cycle that runs until done
🧠
THINK
LLM reasons about the task, current state
📞
ACT
Call a tool with specific parameters
👁️
OBSERVE
Read tool result, check if successful
🔁
REPEAT
Back to Think or exit if done
💡
This IS the Claude Code agent
The entire power of Claude Code is this loop. Each cycle can call different tools based on what was observed. It's autonomous problem-solving.

Smart Model Selection Strategy
Use the right model tier for each type of task to control costs
🌿
Haiku — Sub-tasks
Use for documentation writing, simple transformations, repetitive operations. Ultra cheap per token. Sub-agent tasks.
🎵
Sonnet — Main Agent
Use for complex reasoning, architecture decisions, multi-step planning. The parent agent runs Sonnet 4.6 (latest).
💰
Cost Optimization Strategy
Parent agent uses Sonnet for reasoning. Spawned sub-agents use Haiku for simple tasks. This dramatically reduces total cost without sacrificing output quality.
