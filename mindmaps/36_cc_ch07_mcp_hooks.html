What is MCP — Model Context Protocol
The Anthropic open standard that lets any AI connect to any external tool or data source
★
Open standard, not Claude-only — MCP is published by Anthropic but adopted across the AI industry. Any LLM or agent framework can use it.
✓
What it solves — Claude's built-in tools are fixed. MCP lets you add unlimited custom tools: databases, APIs, browsers, proprietary systems.
✓
How it works — you run an MCP server that exposes tools as functions. Claude calls those functions like any other tool. The server does the actual work.
✓
Configuration — a single JSON config file in your home directory registers all MCP servers. Claude loads them at startup automatically.
✓
Security — MCP servers run isolated from each other. Least-privilege: only expose the tools each agent actually needs.
💡
⭐ The Big Picture
MCP is to AI agents what npm packages are to Node.js. A growing ecosystem of pre-built MCP servers for databases, browsers, APIs, SaaS tools — install and connect in minutes.
⚖️
stdio MCP vs HTTP MCP
Same protocol, two deployment models — choose based on where your tools live
🖥️ stdio MCP (Local)
Runs as a process on your machine
Communicates via stdin/stdout pipes
No network needed — ultra-low latency
Great for: local files, databases, CLI tools
Config: "command": "python mcp.py"
VS
☁️ HTTP MCP (Hosted)
Runs on a remote server or cloud function
Communicates via HTTP requests + SSE
Sharable across a team — one URL for all
Great for: shared APIs, SaaS integrations
Config: "url": "https://mcp.example.com"
📄
MCP Configuration File
One JSON file, registered in ~/.claude/ — loaded automatically at startup
{
  "mcpServers": {
    // stdio — local Python server
    "duckduckgo": {
      "command": "python",
      "args": ["mcp_ddg.py"]
    },
    // HTTP — remote hosted server
    "playwright": {
      "type": "http",
      "url": "https://mcp.playwright.dev"
    }
  }
}

Hook Events — Event-Driven Agent Control
Four lifecycle events you can intercept and run any script against — Python, Bash, or PowerShell
PreToolUse
Before Every Tool Call
Runs before Claude executes any tool. Use to: validate inputs, log the intent, block disallowed operations, inject additional context.
Runs: .py · .sh · .ps1
PostToolUse
After Every Tool Call
Runs after a tool completes. Use to: log results, trigger side effects, validate output quality, send metrics to Datadog or CloudWatch.
Runs: .py · .sh · .ps1
Notification
On Agent Notification
Fires when Claude sends a notification (long task complete, error, needs input). Use to: send Slack/Teams alert, play a sound, send email.
Runs: .py · .sh · .ps1
Stop
On Session End
Fires when the agent session ends. Use to: clean up temp files, commit changes to git, send a completion report, archive logs.
Runs: .py · .sh · .ps1
🪝

MCP & Hooks Pitfalls
What goes wrong when MCP is misconfigured or hooks are abused
!
Too many MCP servers — each server adds latency at startup; 10+ registered servers can noticeably slow Claude Code's first response
!
Broad tool permissions — giving an MCP server write access to production databases is like giving a junior dev the root password. Scope tightly.
!
Heavy hooks — PreToolUse hooks that do expensive work (API calls, DB queries) block every tool call. Keep hooks fast or run async.
!
No error handling in hooks — a crashing hook can kill the entire agent session. Always wrap hook scripts in try/except or set -e safety nets.
🚫 Overpermission 🚫 Heavy Hooks 💡 Scope Tightly
