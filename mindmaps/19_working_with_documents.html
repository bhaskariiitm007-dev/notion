PDF Analysis Workflow
50-page startup report → 5 investor-ready insights
1
Attach PDF — drag and drop or click paperclip; Claude extracts full text automatically
2
Set audience context — "Who is this for?" Claude asks: VC audience? Leadership team? General summary?
3
Set depth — "3–4 lines per insight" or "executive 1-pager" — controls verbosity
4
Get insights — structured bullet analysis, tailored for your stated audience and use case
5
Go visual — "Make a short infographic I can share in my call" — instant shareable HTML output

Common Mistakes with Document Analysis
Avoid these to get sharper, more useful outputs
!
Vague prompt + file — "analyze this" gives nothing useful; always specify the audience and goal
!
Ignoring Claude's questions — when Claude asks for context, give complete answers; the output quality depends on it
!
Stopping at the first output — the first analysis is a draft; iterate to add visuals, change audience framing, deepen specific points
!
Not tracking what Claude is doing — watch the thinking trace; understanding its steps helps you correct it when needed
🚫 Vague Prompts 🚫 Skip Iteration 💡 Watch Traces
