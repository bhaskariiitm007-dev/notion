🏗️
Anatomy of a Great CLAUDE.md
Six sections that make Claude a team-aligned developer
📋 Section 1: Project Overview
What does this project do? Business domain, purpose, audience. Gives Claude the "why" before the "how".
🛠️ Section 2: Tech Stack
Specify every tool: Python 3.11, PySpark, Delta Lake, dbt, Airflow, Docker. Claude won't guess — it'll use exactly what you tell it.
✍️ Section 3: Coding Conventions
Naming case (snake_case/camelCase), function length limits, comment standards, type hints required? Docstrings mandatory?
📁 Section 4: Project Structure
Folder hierarchy, where pipelines live, where models go, where tests belong. Claude places files in the right locations.
🧪 Section 5: Testing Standards
Test framework (pytest, unittest), coverage requirements, what to test and what to mock. Ensures all code ships with tests.
⚠️ Section 6: Special Rules & Anti-Patterns
Things to never do: no hardcoded credentials, no print() in production, always use logging. Red-lines Claude must respect.
