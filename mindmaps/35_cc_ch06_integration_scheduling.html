Onboarding Claude Code to an Existing Project
Using Apache Airflow as the real-world example — works with any codebase
01
Clone the Repository
git clone the project. Navigate into the directory. Don't read anything yet — let Claude Code discover the structure autonomously.
02
Run claude /init
Claude reads every file: DAGs, operators, plugins, requirements.txt, configs. Builds a mental model of the entire project in seconds.
03
CLAUDE.md Auto-Generated
Claude creates a project-specific CLAUDE.md: Airflow version, DAG patterns in use, existing operators, coding conventions it detected.
04
Review and Customise
Add prohibitions, preferred patterns, team standards not visible in the code. CLAUDE.md becomes the source of truth for all future agents.
05
Ask Your First Task
"Create a new DAG that reads from S3, transforms with Polars, and writes to Delta Lake — following our existing patterns." Done in minutes.

Power Slash Commands
In-session commands that change how you interact with Claude Code
💬 /btw
Ask a side question without interrupting the current task. "btw, what's the difference between Polars and Pandas lazy mode?" — Claude answers then continues the task.
🎙️ /voice
Switch to microphone input. Speak your task naturally. Great for long instructions you don't want to type. Claude Code transcribes and acts.
📋 /memory
View, add, or clear items in Claude's persistent memory. What it knows about you and your project that carries across sessions.
⏸️ /pause
Pause a long-running task to review what Claude has done so far. Inspect, redirect, or approve before it continues to the next phase.
