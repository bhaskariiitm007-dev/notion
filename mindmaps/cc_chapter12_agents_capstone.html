Interns with Separate Desks
A sub-agent is like hiring an intern who works at their own desk, with their own computer. They don't share your RAM. You hand them a task, they return a summary. Your main session stays clean.
⚡
Parallel = Multiplied Speed
Three sub-agents working simultaneously on code review, test writing, and documentation generation. All three complete in the time one would take. True parallelism for AI work.
💰
Use Haiku for Sub-Agents
Sub-agents do focused, well-defined tasks. They don't need Opus-level reasoning. Haiku is 10× cheaper, just as good for structured tasks. Opus for orchestrating, Haiku for executing.
🧠
Why Sub-Agents? The Problem They Solve
One big session vs. specialized mini-sessions
❌ Single Session Problems
Context fills up — 200k tokens exhausted
Using Opus for simple code scanning — overkill
Each message re-sends 100k+ token history
Can't multitask — waits for each task
Recency bias → forgets earlier context
→
✅ With Sub-Agents
Own context window — never pollutes yours
Use Haiku for sub-tasks — 10× cheaper
Returns only a compact summary, not full context
Run 3 agents in parallel simultaneously
Focused context = sharper, more accurate results
🧩
Sub-Agent Anatomy — What It Has
A mini Claude with its own everything
🧠
Own Model
Can use Haiku or Sonnet — doesn't have to match the parent. Pick the cheapest model that does the job well.
📋
Own Context
Fully isolated context window. When it's done, only the summary returns to you. No context leak.
🔧
Own Tools
Can have different tool access. Give file-scanner agent only read tools — no write. Controlled access.
📚
Inherit Skills
Can inherit parent's skills or have its own specialized skillset. Bring only what it needs.
🪝
Own Hooks
Can have custom hooks for its specialized context. Different guardrails from the parent session.
🗑️
Terminated by Parent
Parent spawns it, parent terminates it. Not a persistent process — born for a task, dies after.
🏢
Agent Teams — Parallel Specialist Architecture
Manager orchestrates, specialists execute — just like a real team
👑 Parent Claude (Opus) — The Orchestrator
Plans the work · Assigns tasks · Collects summaries · Makes final decisions
🔍
Code Reviewer
Haiku
Scans every file for bugs, anti-patterns, security issues. Returns: list of findings.
🧪
Test Writer
Sonnet
Generates unit tests with edge cases for every function. Returns: test files with coverage summary.
📝
Doc Generator
Haiku
Reads code, writes JSDoc / docstrings, updates README. Returns: documentation summary.
⚠️
CRITICAL: Parallel agents must work on mutually exclusive files
If two sub-agents edit the same file at the same time, you get conflicts. Design your agent tasks to operate on different parts of the codebase. Agent 1 handles auth/, Agent 2 handles api/, Agent 3 handles UI/.

Capstone Project — Job Application Tracker · Built Live
Full-stack productivity app from scratch with Claude Code — 8 phases, zero manual coding
Stack used: React TypeScript SQLite Prisma ORM Node.js API Tailwind CSS
PHASE 01
Project Structure
Scaffolds React + Node.js monorepo, tsconfig, package.json, Prisma init, folder structure.
PHASE 02
Database Schema

PHASE 03
REST API
Creates CRUD endpoints: GET/POST/PUT/DELETE /jobs. JSON responses, error handling, validation.
PHASE 04
Kanban Board UI
React Kanban with columns: Applied → Interview → Offer → Rejected. Drag-and-drop between columns.

Add Job Form
