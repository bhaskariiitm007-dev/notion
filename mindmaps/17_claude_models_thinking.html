The Three Claude Model Tiers
From lightning-fast to deepest thinking — match the model to the task
⚡ Fastest / Cheapest Most Powerful / Deepest ⚡
Haiku → ← Opus
TIER 1 — FAST ⚡
Haiku
Speed First
Fastest response time, lowest token cost. Great for repetitive tasks where speed beats depth.
Best for: Summaries, quick Q&A, drafting
TIER 2 — BALANCED ⚖️
Sonnet
Daily Driver
The everyday workhorse. Balances capability and cost for documents, analysis, and code.
Best for: Documents, research, coding
TIER 3 — MAXIMUM 👑
Opus
Deep Reasoning
Most capable. Handles highly complex, multi-step problems that require deep extended reasoning.
Best for: Complex plans, ambiguous tasks
🧠
Extended Thinking Mode
Claude thinks longer before it responds — more depth, more tokens
🔴 Thinking OFF
Immediate response generation
Fast, efficient, lower token use
Good for clear, well-defined tasks
Standard quality for most requests
🟣 Thinking ON
Claude deliberates before answering
Higher quality for complex problems
Visible thinking process in output
Uses more tokens — use judiciously
⚠️
Token Burn Warning
Unnecessary extended thinking wastes credits. Only enable it for tasks where deeper reasoning genuinely matters — complex plans, ambiguous requirements, multi-step analysis.

Managing Usage & Credits Wisely
Don't run out before your key meetings
✓
Check usage regularly — Settings → Usage shows session % and weekly % consumed
✓
Disable unused connectors — each connected app sends context; 10+ apps drains tokens fast
✓
Plan before meetings — check available credits before a key session or presentation
✓
Match model to task — Haiku for simple tasks; Opus only when complexity truly demands it
✓
Extended thinking selectively — only enable when the problem genuinely needs deep reasoning

Common Model Selection Mistakes

Always using Haiku — too weak for complex analysis; cheap now, frustrating later when output is shallow
