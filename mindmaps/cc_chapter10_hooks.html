Event-Driven Architecture
Hooks fire on events inside Claude's agentic loop — think → create file → run bash → respond. Each step is catchable. Your script runs automatically when the event fires.
🛑
The Emergency Brake
Exit code 2 is absolute. Even if you've given Claude full dangerously-skip-permissions mode, exit 2 from a hook stops the action completely. You shall not pass.
💾
Pure Software Logic
Hooks don't need an LLM. They're plain bash/Python/JS scripts. Deterministic, fast, free. No "maybe Claude will catch this" — it will catch it, every single time.
📡
6 Hook Events — What You Can Intercept
Every critical moment in Claude's session is catchable
SESSION_START
Session begins
Runs when Claude Code launches. Use it to play a notification sound, load context, or log session start time.
USER_PROMPT_SUBMIT
Before Claude reads your message
Intercept prompts before processing. Can sanitize, transform, or block certain messages. Example: replace "destroy" with "review".
PRE_TOOL_USE
Before a tool runs — most powerful
Block dangerous bash commands (rm -rf), log tool calls, enforce permissions. The main security guardrail event.
POST_TOOL_USE
After a tool completes
Auto-format code after writes. Log what changed. Run linters. Send Slack/Telegram notification. Async = run while Claude continues.
SUB_AGENT_START/STOP
When a sub-agent is spawned or ends
Monitor sub-agent lifecycle. Log when specialist agents start/fail. Useful for debugging complex multi-agent workflows.
SESSION_END
Session terminates gracefully
Write a session summary. Clean up temp files. Send a completion notification. Runs on proper exit (not terminal kill).
🚦
Exit Code System — The Access Control
Return the right number and you control what Claude does
0
✅ Success — Allow the action
Hook ran fine. Claude proceeds normally. You can pass JSON output back — Claude will read it. Default happy path.
1
⚠️ Non-blocking Error
Hook failed but Claude continues anyway. Good for optional logging/notification hooks where failure isn't critical.
2
🛑 BLOCKING ERROR — Gandalf Mode
Stops the action completely. No exceptions. Bypasses ALL permission modes including dangerously-skip-permissions. stderr message goes back to Claude as an error. Use for hard security rules that must never be broken.
🔬
3 Use Cases — Automation · Logging · Guardrails
Real hooks Claude Code professionals actually build
⚙️ Automation
Run prettier/eslint after every file write. Send Slack notification when Claude finishes a task. Auto-commit changes with a formatted message. Remove long dashes Claude adds to text automatically.
📋 Logging
Write every bash command to audit.log. Record which files were changed and when. Track token usage per session. Monitor sub-agent costs in production. Full audit trail — who deleted what.
🛡️ Guardrails
Block all rm -rf commands (exit 2). Prevent commits to the main branch. Deny access to sensitive env files. Check for API keys in output before writing. Hard limits that Claude cannot bypass even accidentally.
⚡ async: true — Add this to any hook to run it in parallel while Claude continues working. Perfect for notifications, logging, or formatting that doesn't need to block Claude's workflow.
🔐
Setting Up a Hook — How It Works
Added to settings.json in your .claude folder
// .claude/settings.json
{"hooks": {
"PreToolUse": [{
"matcher": "bash",
"hooks": [{
"type": "command",
"command": "~/.scripts/block-rm.sh"
}]
}]
}}
