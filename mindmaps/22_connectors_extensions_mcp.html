Connectors vs Extensions — The Critical Distinction
Two separate integration systems — mixing them up causes confusion
🌐 Web Connectors
For cloud-hosted web applications
Available in both web & desktop app
Requires OAuth authentication per app
Read and write permissions configurable
Gmail Google Calendar Google Drive Notion Slack + many more
VS
🖥️ Desktop Extensions
For machine-local applications
Only available in Claude Desktop App
Runs via AppleScript or local scripts
Accesses things web can't reach
Apple Notes iMessage PDF Tools Chrome Browser File System Terminal
💡
⭐ THE KEY RULE
Claude on the web cannot read your iMessages or Apple Notes because they live on your device. The Desktop App + Extensions bridges that gap. Web lives on cloud; Desktop can reach your machine.

Permission Level System
Control exactly what Claude can and cannot do with your connected tools
✅
Always Allow
Claude executes this action automatically. Best for read-only, low-risk operations.
👁️
Needs Approval
Claude asks you before acting. Use for write operations like creating calendar events or drafts.
🚫
Blocked
Claude cannot perform this action at all. Use for sensitive data access you want to keep private.
⚠️
Safety-First Approach
Claude can make mistakes with write access. Start with "Needs Approval" for all write operations — creating events, deleting files, sending drafts. Never give blind write access to critical tools.

Token Efficiency with Connectors
More connections isn't always better — be selective
🎯 Selective Enabling
Only enable connectors you'll actually use in this conversation. Each connected app sends context tokens.
📊 10+ Apps = Token Drain
Having 10+ connectors active simultaneously drains your free limit much faster than needed.
⚙️ Disable for Sessions
Turn off apps not needed for a task from the chat toolbar. Re-enable when that tool is needed again.
📅 Pre-Meeting Check
Before an important session, verify your weekly limits haven't run low — check Settings → Usage.

Security Best Practices with Connectors
Claude is powerful but can make mistakes — protect your data
!
Always read OAuth permissions — don't click "continue" without understanding what access you're granting to Claude
!
Start with "Needs Approval" for writes — don't give blind write access to Gmail or Calendar on day one
!
Claude can delete your files — if you give file-system write access, one wrong prompt could trigger destructive actions
!
Don't overpromise to clients — "Claude can connect to Gmail" doesn't mean it can do everything; read the actual permission scope
✓
Custom MCP for full control — build your own MCP server to get exactly the permissions you need — no more, no less
⚠️ Read Permissions 🚫 Careful with Writes ✅ Custom MCP = Best Control
