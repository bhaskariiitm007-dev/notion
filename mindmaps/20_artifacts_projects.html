The Instruction Hierarchy — Who Wins When They Conflict?
Three layers of instructions, each more specific than the last — most specific always wins
1
Global Claude Instructions
Set in your account settings. Applies to every single conversation and project. E.g. "Always ask me for goal, context, and action before responding."
Always active
2
Project Instructions (sit on top of global)
Set per project. Applies to all chats within that project. E.g. "This is a US client project. Always be formal. Use brand guideline file for tone."
Project-scoped
3
Chat Instructions (most specific — wins locally)
Set inside a specific chat. Only applies to that conversation. E.g. "For this chat only: respond in Hindi." — Overrides global and project instructions for this chat.
Chat-only
⭐
The Hierarchy Rule to Remember
Global applies everywhere. Project overrides global for its chats. Chat overrides both for itself. Most specific instruction always wins. This is critical to understand for Claude Code too.
