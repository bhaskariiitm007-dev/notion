⭐ Why Isolation Matters
Each sub-agent has its own context window. The Data Analyst doesn't see the Code Writer's file history. This prevents context pollution and keeps each agent sharp at its specific role — quality through focus.
⚙️
Configuring a Sub-Agent
Four dimensions you control when spawning a child agent
🔧 Tool Restriction
Grant only the tools the sub-agent needs. A reviewer gets read_file and grep — never write_file. Principle of least privilege applied to agents.
💾 Memory Scope
Decide what the sub-agent knows about: just the current file, the whole repo, or a summary injected by the parent. Isolated by default.
📚 Skills Allocation
Each sub-agent gets only the skills relevant to its role. The data analyst gets stats skills; the code writer gets formatting skills.
🤖 Model Choice
Heavy reasoning agents get Sonnet or Opus. Lightweight reviewers and formatters can run on Haiku — massive cost savings in large teams.

Sub-Agent vs Skill — What's the Difference?
Both delegate work — but in fundamentally different ways
📚
Skill — a playbook (markdown file). The same parent agent follows the steps. No new context window. No new model call. Just structured instructions.
🤖
Sub-agent — a fully separate agent instance. Its own context, its own tools, its own model choice. Runs in parallel with the parent.
🔑
Rule of thumb — use a skill when the task is within one specialisation; use a sub-agent when the task needs a different context or tool set
📚 Skill = Playbook 🤖 Sub-Agent = Instance

Real Example: Code Review Agent
How to spawn a specialist reviewer as a sub-agent after every code write
1
Parent writes code — using write_file tool; creates new_module.py in the project
2
Parent spawns reviewer — creates sub-agent with only read_file + grep; injects review_checklist.md skill
3
Reviewer runs — reads the file, checks against checklist: type hints, docstrings, error handling, test coverage
4
Reviewer reports — sends structured feedback back to parent: "Line 42: missing error handling for HTTP 429"
5
Parent fixes — acts on feedback; loops until reviewer approves. Ship only reviewed, quality code.
